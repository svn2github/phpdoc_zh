3.2. CVS 的登录

现在您已经有 CVS 模块的写权限了，我们还需要在您的 .cvsrc 加上一些变量。例如增加 CVSROOT 环境变量以便登录 CVS 资源库。

如果在您的根目录中尚无 .cvsrc 文件，那么请建立该文件并加入以下几行内容。注意如果您使用的是 Cygwin，您的根目录即是您运行 bash 外壳时最先打开的目录（即 Cygwin 安装目录下的 home/username目录）。 

cvs -z3
update -d -P
checkout -P
diff -u

每次您向 CVS 资源库提交要执行的操作时，都需要以如下方式指明资源库的 cvs 根目录： 

$ cvs -d :pserver:username@cvs.php.net:/repository checkout phpdoc

您可以设置 CVSROOT 环境变量，以免每次向资源库执行命令时都要指明其根目录。

对于 sh 和 bash 的用户，请将下列内容加入到您的 .bashrc 或者 .profile 文件中： 

CVSROOT=:pserver:username@cvs.php.net:/repository
export CVSROOT

对于 csh 和 tcsh 的用户，请将下列内容加入到您的 .cshrc 或者 .tcshrc 文件中： 

setenv CVSROOT :pserver:username@cvs.php.net:/repository

注意： CVSROOT 环境变量仅在您下次登录的时候才会生效。同时，请不要忘了将 username 替换成您自己的 CVS 账号名。

如果您使用的是 Windows 系统，请用您通常的方法添加环境变量。在 Windows 9x 系统下，使用 set 命令，在 Windows 2000 系统下使用环境变量的设置对话框。

您已经完成了所有的准备工作，请使用如下命令登录到 CVS 资源库： 

$ cvs login

或者，如果您还没有设置 CVSROOT 环境变量，为了保证您访问了正确的 CVS 资源库，请使用如下命令： 

$ cvs -d :pserver:username@cvs.php.net:/repository login

请将以上命令行中的 username 替换成您自己的 CVS 用户名。您将被要求提供您的密码。一旦您成功的登录到了 CVS 资源库，您的密码将被加密后保存在您的根目录的 .cvspass 文件中，这样，您以后就不用每次在提交命令前都要进行登录操作，除非您删除了该文件，或者执行了 cvs logout 命令。该文件的内容将不会改变，并一直保持您的登录状态。
