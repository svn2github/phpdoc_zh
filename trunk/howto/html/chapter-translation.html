目前 PHP 文档存在很多语种的翻译版本。有些语种是由一组翻译人员来维护的（如德语），有些则是由一个人完成的（如日语）。作为一个翻译者需要知道很多东西，尽管它们都十分简单。

9.1. 开始一个新语种的翻译工程

目前开始一个新语种的翻译需要如下几个步骤：

  • 请查询邮件列表，确认该翻译工程是否已经在进行。如果是，请跳过以下步骤，并通过该语言翻译工程的邮件列表与其它正在进行此项工作的翻译者取得联系。【译者注：目前 PHP 手册简体中文版的翻译工程已经建立并已经在进行中。如果您对此翻译工作感兴趣，或者希望加入到我们的小组开展翻译工作，请发电子邮件到 <doc-zh@lists.php.net> 了解情况。】

  • 申请一个 CVS 账号。在申请时，请说明您希望开始一个新语种的翻译工作。

  • 当您得到您的 CVS 账号以后，请发电子邮件到 systems@php.net 要求为新的翻译工程建立一个新的 phpdoc-LANGCODE CVS 模块，以及一个新的 doc-LANGCODE 邮件列表。

  • 按 CVS 一章中所描述的方法下载 phpdoc 和 phpdoc-LANGCODE 模块（在实际下载时，需要用phpdoc-LANGCODE-dir 来指代您的翻译模块）。

  • 将 en/language-defs.ent 和 en/language-snippets.ent 两个文件拷贝到新建立的 phpdoc-LANGCODE 模块的根目录下（也就是您文件系统中的 phpdoc/LANGCODE 目录），并翻译这两个文件的内容。

  • 拷贝英文版本目录下的所有文件并开始翻译它们（请不要上传未翻译的文件），每次您新增一个文件，都需要重新运行 configure 配置脚本。请查阅“修订版本跟踪”以获取关于如何减轻您在跟踪英文版本和您所翻译文件的版本时工作的信息。

以下是在建立一个新的翻译工程前需要慎重考虑的几个重要问题：

  • 您是否能够独自建立并完成整个手册的翻译工程？如果不能，您是否能够建立一个团队（我们也建议以这种方式）来完成该语种的翻译工程？

  • 您是否确认您的语言和文字（包括字母、字形或者单字）已经被 DocBook 样式表单所支持？如果您不清楚，请到邮件列表询问。
