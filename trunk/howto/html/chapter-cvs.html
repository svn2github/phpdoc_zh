PHP 文档的维护工作是借助 CVS (Concurrent Versions System，版本协调系统)来完成的。CVS 的使用让文档工作者在不需要等待 其他人的工作的情况下同时更新 PHP 文档不同的文件。

一个 CVS 系统包含有一个存有所有文件的中央服务器。一个 CVS 服务器可以作为许多不同的资源库 的服务器，PHP 文档资源库的名称是 phpdoc。要访问该服务器，您需要在 您的系统上安装一个 CVS 的客户端程序。

当您决定开始要对一个文件开展工作时，您需要先校验（即下载）该文件。然后才能在您下载的文件的 基础上进行修改。当您完成后，您需要提交您所做的修改，即上传该文件。CVS 客户端程序会要求您 输入简短的注释，您可以在这里输入您所做的修改的概括性描述。CVS 服务器将储存所有更新过的文件 的历史记录，包括它们所附带的注释。这样，其他人就能够知道您对那个文件作了哪些修改，因为在您 向 CVS 服务器提交您的结果时，系统会自动比较最新版本和上一个版本的差别，并将其内容自动发送 到 邮件列表，其结果可以通过 CVS 客户端来查 看，或者通过 WEB 界面：http://cvs.php.net/。您同样可 以根据需要利用您的 CVS 客户端在 CVS 服务器上删除或者增添文件。

本节内容并非希望成为 CVS 使用指南，仅仅只是引导您能够尽快的学会下载 phpdoc 目录树以及提交您的更新。如果您需要 CVS 完整的文档，请查阅 http://www.cvshome.org/。

CVS 使用指南可以在以下地址找到：

  • http://www.arc.unm.edu/~rsahu/cvs.html

  • http://cellworks.washington.edu/pub/docs/cvs/tutorial/cvs_tutorial_1.html

您也可以随时在您的 CVS 客户端中使用 man cvs 命令来获取帮助。该命令 将得到被称为“manpage”的帮助文档。

3.1. 如何获取 CVS 账号

首先，在您能够进行文档有关工作之前，您需要先访问 cvs.php.net 的资源库。您随时都可以匿名的下载资源库的模块，但是要上传文件则必须 有 CVS 账号。如果您仅需要下载文件，您可以使用只读的账号和密码。请查阅 http://www.php.net/anoncvs.php 以获取更多信息。

同时也请注意，即使您在我们的服务端有 CVS 的账号，您也可能没有 phpdoc 模块的写权限。我们的 Karma 系统控制着不同的账号访问不同资源库的权限，所以如果您有 CVS 账号，但是没有访问 phpdoc的 Karma 权限，请向 Karma group@php.net 垂询。注意，有 PHP 源 码目录树 Karma 写权限的人，同样也有 phpdoc 的 Karma 权限。

在 php.net 获取一个 CVS 账号并不困难，您只需完成以下两件事：

  • 用英文向 group@php.net发 送电子邮件，说明您希望利用 CVS 账号来做些什么（即参与 PHP 文档的维护工作）。同时也请 付上您的一些个人信息和背景等以便开发人员了解您来自何处。

  • 一旦开发小组回应了您的电子邮件，您需要访问 http://www.php.net/cvs-php.php 来填写正式的注册表单。

以上两个步骤以及和所有 cvs.php.net 的 CVS 账号有关的信息，都可以在注册表单所在的网页找 到。其地址为 http://www.php.net/cvs-php.php。注 意，利用该注册表单注册并非是自动处理的过程。如果您在提交表单后的几天内没有收到任何回应，请 别忘了再次提交您的申请。
