2.3. XML 样式表单相关工具

为了能够成功的使用 XML 样式表单，您必须有一种 XSLT 处理器。这些已经足够用来生成文档的 HTML 版本，如果您还希望能够生成一个方便打印的版本，则您还需要一个 FO 处理器。

这是一个不取决于操作系统的部分，因为大部分的工具都是用 Java 语言来开发的，或至少提供 Java 版本。这也就是说，您可以在任何具有 Java 虚拟机的操作系统上使用它们。请访问 Java 的主页以获取更多信息。

注意： 通过 XSL 样式表单来使用 phpdoc 的 XML 源文档生成各种不同格式的版本，目前是可选的。这个方法目前并不像 DSSSL 样式表单一样已经被良好的支持，但是它是一种很有前景的技术。如果您不希望使用 XSL 样式表单的话，您并不需要安装这里提及的任何工具。尽管如此，我们计划在未来的 HTML 版本，甚至可能在 PDF 的生成工作中使用 XSL 技术，如果我们最终能够使之正确的工作的话。

PHPDOC 的 Makefile 文件有使用 XSL 样式表来生成文件的选项，但目前还处于实验阶段（而且还需要 xslproc）。在运行 ./configure 以后，您可以运行 make html_xsl 命令来用 XSL 样式表生成 HTML 文件。

XSLT 处理器：

  • Saxon: http://saxon.sourceforge.net/

  • Xalan: http://xml.apache.org/xalan-j/

  • Gnome LibXSLT（也被称作 xsltproc）: http://xmlsoft.org/，Windows 版本：http://www.fh-frankfurt.de/~igor/projects/libxml/index.html，该处理器也可由最新的 cygwin 安装。

注意： 我们已经做过测试，其它的 XSLT 处理器，例如 XT 和 MSXML 等，无法处理 DocBook 的 XSL 样式表，因此它们无法用在即使是最简单的 HTML 生成工作中。

FO 处理器：

  • PassiveTeX: http://users.ox.ac.uk/~rathz/passivetex/

  • FOP: http://xml.apache.org/fop/

  • XEP: http://www.renderx.com/
